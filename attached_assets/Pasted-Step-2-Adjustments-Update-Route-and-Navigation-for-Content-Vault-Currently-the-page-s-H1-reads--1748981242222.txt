---

## 1. Rename the Page File to ContentVault.tsx

1. In `client/src/pages/`, **rename** `Templates.tsx` → `ContentVault.tsx`.  
   - This will change the Next.js (or React Router) route from `/templates` to `/content-vault`.  
   - Remove or archive any old `ContentVault.tsx` file if it still exists, to avoid duplicates.

2. Update the imports and references inside the app:  
   - In `App.tsx` (or wherever your routes are defined), change any `<Route path="/templates" component={Templates} />` to `<Route path="/content-vault" component={ContentVault} />`.  
   - If using Next.js, no `App.tsx` change is needed—Next’s file-based routing will pick up `/content-vault` automatically once the file is renamed.

3. Within `ContentVault.tsx`, **update any internal links** or breadcrumbs that read `/templates` to now use `/content-vault` (e.g., `<Link to="/templates">` → `<Link to="/content-vault">`).
