We need to turn each slide into a true, touch-friendly horizontal carousel of feature icons, with pop-up controls, so users can pick a filter or adjustment without losing sight of the preview. Here’s exactly what to do:

1. Replace the vertical grid of filter buttons with a horizontal, swipeable carousel

On mobile (and tablet), display the Vogue filters as a single row of large, square black buttons (each 80×80 px) aligned left-to-right, inside a container with overflow-x: auto; display: flex; and no vertical scrollbar.

Each filter button should show a minimal white icon or text label (e.g., “Glow,” “Luna,” etc.) centered, using Prata font.

Below that row, reserve a small “Intensity” pill (e.g., “Intensity: 65%”) that only appears when a filter is actively selected; it should slide up from under the filter carousel as a semi-transparent black overlay with white text.

Swiping left/right anywhere on the filter button row should scroll through all 10 filters. Add subtle left/right chevron overlays at 10 % from each edge (semi-transparent black with white chevrons) that only appear while the user is touching or hovering.

Tapping a filter button selects that filter and immediately applies it to the Preview. The “Intensity” pill appears. If the user taps “Intensity” or the pill’s numeric value, a small horizontal slider pops up (black track, white thumb) that lets them adjust from 0–100 % in real time. Once they tap outside the slider, it collapses and leaves the chosen intensity.

On desktop (≥ 768 px), the existing grid of filters can remain as is (two columns), but wrap it in a fixed‐height container so it never pushes the Preview below the fold on narrower browser windows.

2. Turn the vertical “Adjust” sliders into a horizontal swipe carousel of adjustment icons

Instead of stacking “Brightness,” “Contrast,” “Vignette,” etc. vertically, replace that entire block on mobile with a single horizontal row of black icon buttons (each 80×80 px) representing each adjustment (sun icon for brightness, circle for contrast, droplet for saturation, thermometer for temperature, and a vignette icon for edges).

Render this row inside a container with overflow-x: auto; display: flex; so users can swipe through all adjustment icons. Add the same semi-transparent chevron overlays on the far left/right that appear when they touch or hover.

When the user taps on an adjustment icon (for example, the sun), open a small pop-up “Adjust Brightness” panel directly above that icon:

Show a mini vertical slider (black track, white thumb) for that specific adjustment, labeled “Brightness: 35%.”

The user can drag it up/down to update the image in real time.

Tapping outside the slider or hitting a close “✕” icon in its top-right collapses that panel.

Beneath the carousel of icons, keep the “Reset All” and “Download” buttons (black background, white text), always visible.

On desktop, the adjustment sliders remain visible side-by-side under the “Adjust” tab as before; only on screens narrower than 768 px do you switch to this icon-based carousel + pop-up sliders approach.

3. Ensure the Preview always remains at the very top, with no vertical scrolling required to adjust filters or sliders

4. Responsive breakpoints and style consistency

Use a media query at max-width: 767px to switch to the above mobile carousel layout. On larger screens, keep the existing two-column desktop view (filters and adjustments side-by-side under the Preview).

Maintain the luxury Vogue styling: black backgrounds, white Prata labels, subtle 2 px white borders around buttons, thin charcoal shadows on active states, and the same hover/focus transitions you’ve already implemented.

For the swipe functionality, use react-swipeable. Attach its onSwipedLeft and onSwipedRight handlers to move between the two slides (“Filters” vs. “Adjust”). Add aria-label to each slide for accessibility (“Filters carousel slide” and “Adjust carousel slide”).
