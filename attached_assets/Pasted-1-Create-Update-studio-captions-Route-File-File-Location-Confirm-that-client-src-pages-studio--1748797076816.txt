5. Styling & Theming
Throughout both desktop and mobile:

Use Prata Serif for all headings, labels, and body text.

Black (#000) backgrounds for primary buttons, toggles, and highlighted states.

White (#FFF) text on black; Charcoal (#4A4A4A) for secondary text.

Light Gray (#F5F5F5) backgrounds inside cards and inputs.

Gold Accent (#C19A6B) for any hover outlines or active borders on selected cards (subtle!).

Maintain 4px border-radius on all inputs, buttons, and cards for a refined look.

Use consistent spacing: 16px vertical gaps between form fields; 24px between main sections.

Ensure responsive breakpoints: 0–375px (small phones), 376–767px (tablets), ≥ 768px (desktop).

Add smooth transitions (transition: all 0.2s ease-in-out) for hover states on buttons and toggles.

6. Accessibility & Keyboard Support
Dropdown & Textarea: Ensure they have aria-label attributes (e.g., aria-label="Choose Caption Template").

Tone Toggle Pills: Add role="radio" and aria-checked="true/false" per pill. Enable arrow-key navigation:

Left/right arrows move focus between pills.

Generate Button: aria-live="polite" after generation so screen readers announce “Caption generated.”

Caption Cards & Copy Button:

Each card: role="button" tabindex="0" so users can “Select” via Enter/Space.

“Copy” icon: aria-label="Copy caption to clipboard" and ensure it is keyboard-focusable.
